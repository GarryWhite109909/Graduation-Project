# 项目过程记录

## 2026-06-29 实验修复与重跑

### 背景

已完成 exp_01 / exp_02 / exp_03 三个实验，但在评审中发现严重问题：14 个样本源码里存在 `# 期望/漏洞/安全` 等注释，直接泄露了答案，导致 100% 准确率无法证明模型具备真实语义理解能力。同时规划改用 `gemma4:12b` 作为主要实验模型。

### 本次工作内容

1. **删除样本答案泄露注释**
   - 遍历 `experiments/exp_01_basic_scan/samples/` 下 14 个文件
   - 删除所有包含 `期望`、`漏洞`、`安全`、`样本` 的顶部与行间注释
   - 保留功能代码本身不变
   - 更新 `manifest.json` 描述为"无答案泄露版本"

2. **切换主实验模型为 gemma4:12b**
   - 修改 `src/llm_client.py` 默认模型
   - 修改 `experiments/exp_01_basic_scan/run_experiment.py` `--model` 默认值
   - 修改 `experiments/exp_03_rag_knowledge/run_rag_experiment.py` `--model` 默认值
   - 修改 `experiments/exp_03_rag_knowledge/knowledge_data/test_rag.py` 默认模型
   - 更新 `README.md`、`规划.md`、`docs/临时提示词_下一步计划.md` 中的模型说明

3. **修复 Ollama keep_alive 兼容性问题**
   - 现象：`ollama generate` 传 `keep_alive="-1"` 返回 400：`time: missing unit in duration "-1"`
   - 原因：新版 Ollama 不再接受无单位字符串，但整数 `-1` / `0` 仍可用
   - 修复：在 `src/llm_client.py` 的 `generate()` 中加入 `_normalize_keep_alive()`，把字符串 `"-1"`/`"0"` 自动转整数
   - 影响：所有历史调用 `keep_alive="-1"` 的脚本无需再改，由 llm_client 统一兼容

4. **修复 Semgrep 运行环境问题**
   - 现象：graproj conda 环境里 `semgrep --version` 报错 `ImportError: cannot import name 'LogData' from 'opentelemetry.sdk._logs'`
   - 原因：opentelemetry 相关包版本与 semgrep 1.168.0 不兼容
   - 修复：`pip install --upgrade opentelemetry-sdk opentelemetry-exporter-otlp`
   - 验证：Semgrep 成功跑完 14 个样本

5. **备份旧结果并重新运行三个实验**
   - 将旧 `results.json` 备份为 `results.gemma4-26b-with-leaked-comments.20260629-130538.json`
   - 用 `gemma4:12b` 重跑 exp_01
   - 用 graproj conda 环境重跑 exp_02（Bandit / Semgrep）
   - 用 `gemma4:12b` 重跑 exp_03（RAG+LLM）

### 关键结论

- 删除答案泄露注释后，`gemma4:12b` 仍保持 100% 准确率，说明模型具备真实语义理解能力，不是靠注释作弊。
- `gemma4:12b` 在典型样本上表现与旧 `gemma4:26b` 相当，但平均耗时从 56.7s 降到 45.24s，更适合作为主力模型。
- RAG 检索质量良好（Top-1 类型命中率约 85.7%，Top-3 覆盖率 100%），且不引入误报、不显著增加耗时。
- 当前处于"准盲测"状态：prompt 和源码中均无答案，但文件名（如 `sql_injection_01.py`）仍可能泄露漏洞类型。后续若要求严格盲测，可对文件名做中性化处理。

### 踩坑记录

1. **Ollama 版本兼容性**
   - 旧脚本习惯 `keep_alive="-1"`，新版 Ollama 会 400
   - 最佳实践：llm_client 统一做规范化，外部调用仍可用字符串

2. **conda 环境隔离**
   - 默认 shell 用的是系统 Python，找不到 `semgrep`/`chromadb`
   - 批量跑实验必须先 `conda activate graproj`
   - `run_baseline.py` 虽然会把当前 Python 的 bin 目录加入 PATH，但 semgrep 依赖的 opentelemetry 仍需在 conda 环境里正确安装

3. **HuggingFace 离线模型加载**
   - `build_knowledge.py` 与 ChromaManager 启动时会尝试联网检查 `all-MiniLM-L6-v2`
   - 模型已缓存在 `~/.cache/huggingface/hub/`
   - 跑 RAG 实验时应设置 `HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1` 避免网络抖动导致失败

4. **JSON 结果文件体积大**
   - 每个 results.json 包含 raw_output 完整文本，60-80KB
   - 备份的旧结果文件更大，已加入 `.gitignore` 避免入库
