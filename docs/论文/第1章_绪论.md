# 第1章 绪论

## 1.1 研究背景

软件安全漏洞造成的经济损失与社会影响持续扩大。IBM《2024 年数据泄露成本报告》显示,全球单次数据泄露事件平均损失达到 488 万美元,较 2023 年增长 10%,为历年最大增幅;其中由系统漏洞利用引发的泄露事件占比持续位于前列。中国国家互联网应急中心(CNCERT)发布的《2024 年上半年我国互联网安全态势报告》指出,上半年收录的通用软硬件漏洞数量同比增长 17.9%,Web 应用层漏洞仍为攻击者主要入口。在漏洞数量持续增长的背景下,代码安全审计在软件开发生命周期中的位置前移,成为 DevSecOps 实践的关键环节。

工业界长期依赖基于规则与数据流的静态应用安全测试(SAST)工具,如 Bandit、Semgrep、CodeQL 等。这类工具的核心机制是模式匹配与污点流分析,对已知漏洞模式(CWE-89 SQL 注入、CWE-78 命令注入、CWE-22 路径穿越等)检出速度快、误报可控,在 CI/CD 流水线中表现出良好的工程稳定性。然而,规则匹配的边界在面对复杂业务逻辑、绕过式过滤、跨函数污点流以及非典型漏洞变体时迅速收紧。本研究的对比实验(详见第 4.2 节)显示,在一段使用 `os.path.join` 拼接用户输入的路径穿越样本上,Bandit 与 Semgrep 均未报错,而 7B 参数规模的本地大语言模型 Qwen2.5-Coder-7B 正确识别出 `os.path.join` 不构成安全边界、攻击者可通过 `../` 实现目录穿越的风险。规则工具对"看似标准库推荐用法实则可绕过"的场景缺乏语义层判断,这一局限难以通过扩充规则集根本性解决。

大语言模型(Large Language Model, LLM)的兴起为代码安全审计提供了新的技术路径。GPT-4、Claude 3、Gemini 1.5 等闭源模型在 SWE-bench、BigVul 等代码安全评测集上展现出接近人类安全工程师的判定能力,但其闭源、按 token 计费、数据需上传云端的使用模式,与代码作为核心资产对保密性、合规性、可审计性的要求形成冲突。开源代码大模型 Qwen2.5-Coder、DeepSeek-Coder、CodeLlama、StarCoder2 等的发布,使得在本地 GPU 上部署具备代码理解能力的 LLM 成为可能,但通用代码模型在安全领域的零样本表现仍存在三类瓶颈:一是漏洞模式知识盲区,如将参数化查询的 `?` 占位符误判为字符串拼接;二是判定边界过度敏感,对安全对照样本产生幻觉式误报;三是 CWE 分类能力薄弱,常将路径穿越错标为命令注入、将反序列化错标为命令注入。直接将通用代码模型用于安全审计,召回率与误报率均无法满足工程要求。

如何在不依赖闭源 API 的前提下,通过领域知识注入与推理分布校准将通用代码大模型转化为网络安全专用模型,并在消费级 GPU(16 GB 显存)上完成训练与推理,是本研究要回答的核心问题。

## 1.2 研究意义

### 1.2.1 学术意义

本研究在 PEFT(Parameter-Efficient Fine-Tuning)方法体系内探索"知识注入"与"风格微调"的边界。当前主流的 LoRA、QLoRA、rsLoRA、DoRA 等方法主要面向指令跟随与下游任务适配,默认假设是增大适配器容量即可承载更多任务知识。本研究的 Phase 2 实验在 7B 强基座上证实:将 LoRA rank 从 8 增至 32 并配合 rsLoRA 与双倍训练轮数,严格召回率仅提升 1.6 个百分点,而误报率反而上升 19.2 个百分点,CWE 错标数完全未变(32 类)。这一反直觉结果表明,在已经具备较强代码理解能力的高基座模型上,单纯增大适配器容量并不等价于知识注入,LoRA 在此场景下更多承担输出风格校准而非知识获取的角色。该结论为后续 PEFT 研究中"何时需要 CPT、何时 LoRA 足够"的方法选择提供了实证依据。

本研究在 7B 模型上落地了 KnItLM(Knowledge Injection Through Language Model)的继续预训练范式。KnItLM 的核心思路是先在 base 模型上做领域语料的 causal LM 继续预训练,再将 LoRA adapter 合并回 Instruct 基座,以此在保留指令跟随能力的同时注入领域语言模式。本研究在 1400 条 CVE/CWE/OWASP 语料上完成 CPT 训练,严格召回率(CWE 对齐口径)从 41.0% 提升至 63.9%,误报率从 11.5% 下降至 3.8%,CWE 错标数从 32 减少至 16,证明 CPT 路线在 7B 规模上具备可行性,为中小型机构在算力受限条件下构建领域专用模型提供了可复现的工程路径。

本研究进一步以 Qwen3-Coder-30B 为 teacher 模型,通过 Prompt Distillation 修复 CPT 引入的参数化查询幻觉副作用,探索"知识注入 → 分布校准"两阶段训练范式在代码安全任务上的适用性。

### 1.2.2 工程意义

本研究的工程价值体现在三个方面。第一,系统在 AMD Radeon RX 9060 XT 16 GB 消费级显卡上完成全部训练与推理,通过 QLoRA 4-bit 量化、rsLoRA、AOTRITON attention、TunableOp 离线调优等技术组合,将 7B 模型的训练显存占用压缩到 16 GB 以内,单步训练时间较未优化基线下降约 58%。这一结果表明,本地化部署的网络安全专用模型不必依赖数据中心级算力,中小型安全团队具备自建能力。

第二,系统输出的判定结果包含自然语言解释、漏洞类型(CWE 编号)、source-sink 污点路径与可执行修复代码,而非传统工具的"漏洞类型 + 规则编号 + 行号"三元组。在人工审计环节,审计员对单条告警的理解时间从逐条核对规则文档的 30 分钟级降低到阅读一段自然语言解释的 5 分钟级,这一差异在漏洞数量大、修复时间紧的场景下具有实质性意义。

第三,系统的训练数据、训练脚本、评估脚本、实验报告、模型 adapter 均以可复现的方式开源,实验结果文件按时间戳命名保留历史版本,评估口径同时给出单次口径与多数表决口径,并采用 Wilson score 区间给出 95% 置信区间。这一工程规范为后续研究提供了可比对的基线。

## 1.3 研究问题与目标

### 1.3.1 研究问题

本研究围绕以下四个递进问题展开:

**问题 1**:在不做任何训练的前提下,本地部署的开源 7B 代码大模型能否完成代码漏洞检测任务?其能力边界在哪里?

**问题 2**:RAG 知识库增强、AST 代码切片、Prompt 工程等无训练手段能在多大程度上提升零样本推理的判定质量?这些手段的能力天花板在哪里?

**问题 3**:在已经具备较强代码理解能力的 7B 强基座上,通过增大 LoRA 适配器容量(rank 从 8 提升至 32)能否注入额外的网络安全领域知识?如果不能,根本原因是什么?

**问题 4**:继续预训练(CPT)能否真正实现知识注入?CPT 引入的副作用(如参数化查询幻觉)能否通过 Prompt Distillation 修复?最终模型在严格召回率、误报率、CWE 分类准确率上能否同时优于零样本基线?

### 1.3.2 研究目标

本研究旨在构建一个本地部署、领域知识注入、可持续改进的代码安全审计系统,具体目标如下:

1. 在 87 段覆盖 34 类 CWE 的多语言代码样本上,建立零样本推理基线,量化 LLM 与传统 SAST 工具的能力差异;
2. 通过 RAG 消融对照(四组:RAG+LLM / 纯 LLM / 随机知识 / 无关文本)、Top-K 对比、Prompt 工程消融(五种策略)系统评估无训练手段的能力边界;
3. 通过 Phase 1~3 三阶段训练实验(lr sweep / r=32 增容 / KnItLM CPT)厘清"风格微调"与"知识注入"的边界,在 7B 模型上实现网络安全领域知识注入;
4. 通过 Phase 4 Prompt Distillation 与 Phase 5 DPO 进行推理分布校准,修复 CPT 副作用并进一步降低误报率;
5. 全流程在 16 GB 显存的消费级 GPU 上完成,训练与推理代码、数据、adapter、评估结果均以可复现方式提供。

## 1.4 主要贡献

本研究的主要贡献概括为以下五点。

**贡献 1:完整的零样本推理基线与能力边界量化。** 在 87 段 v3 修复后样本(覆盖 34 类 CWE、4 种编程语言)上完成 5 个实验(exp_01~exp_05),系统量化了 Qwen2.5-Coder-7B 的零样本能力:典型样本准确率 100%,难样本多数表决准确率 78.2%,CoT Prompt 策略召回率 95% 为最优单一策略。通过 RAG 四组消融对照实验证明,在修复答案泄露后,RAG 未带来检测能力提升,模型基座已掌握典型漏洞模式,RAG 的价值更多体现在可解释性与知识可扩展性。

**贡献 2:"LoRA 增容 ≠ 知识注入"的反直觉实证。** 在 7B 强基座上完成 Phase 1 lr × rsLoRA 6 组网格搜索与 Phase 2 r=32 + rsLoRA + e=2 增容实验。Phase 2 结果显示,严格召回率仅提升 1.6 个百分点,误报率反而上升 19.2 个百分点,CWE 错标数完全未变。这一结论打破了"增大 LoRA 容量即可承载更多知识"的默认假设,为 PEFT 方法选择提供了实证依据。

**贡献 3:KnItLM CPT 在 7B 模型上的知识注入突破。** 在 Qwen2.5-Coder-7B-Base 上使用 1400 条 CVE/CWE/OWASP 语料完成 CPT(LoRA r=64, rsLoRA, 1 epoch, lr=2e-5),合并到 Instruct 基座后,严格召回率从 41.0% 提升至 63.9%(+23pp),误报率从 11.5% 下降至 3.8%(-7.7pp),CWE 错标数从 32 减少至 16,幻觉率从 56.1% 下降至 29.1%。这是本研究的核心突破,证明 CPT 路线在 7B 规模上能够实现真正的领域知识注入。

**贡献 4:Prompt Distillation 修复 CPT 副作用的两阶段训练范式。** Phase 3 的 CPT 引入了 4 个参数化查询幻觉回归样本(Phase 1 答对、Phase 3 答错),表现为模型将参数化查询误判为字符串拼接漏洞。本研究以 Qwen3-Coder-30B(MoE)为 teacher,通过 Prompt Distillation(KL 散度损失,α=0.5, T=2.0)在 KnItLM student 上校准推理分布,目标是在不损失 Phase 3 召回率的前提下修复幻觉副作用。该两阶段训练范式("知识注入 → 分布校准")为后续领域专用模型训练提供了方法论参考。

**贡献 5:消费级 GPU 上的完整训练流水线工程实践。** 在 AMD Radeon RX 9060 XT 16 GB 显卡上,通过 QLoRA 4-bit NF4 量化、rsLoRA(缩放因子 1/√r)、DoRA(magnitude+direction 分解)、AOTRITON Triton Flash Attention、TunableOp 离线 GEMM 调优等技术组合,将 7B 模型的训练显存压缩到 16 GB 以内,单步训练时间下降约 58%。整套训练-评估-对比流水线以 Shell + Python 脚本形式提供,实验结果按时间戳命名保留历史版本,支持多种子评估与 Bootstrap 显著性检验。

## 1.5 论文组织结构

本论文共分六章,组织结构如下。

**第 1 章 绪论。** 阐述研究背景、研究意义、研究问题与目标、主要贡献与论文组织结构。

**第 2 章 相关工作。** 从三个角度梳理文献:传统静态应用安全测试工具(Bandit、Semgrep、CodeQL)的能力与局限;大语言模型在代码安全审计中的应用现状(零样本推理、RAG 增强、微调路径);参数高效微调与知识注入方法(LoRA、QLoRA、rsLoRA、DoRA、CPT、Prompt Distillation、DPO)。

**第 3 章 研究方法与系统设计。** 介绍系统的四层架构:数据层(CVE/CWE/OWASP 语料、CoT 蒸馏样本、87 段测试集)、训练层(QLoRA SFT、KnItLM CPT、Prompt Distillation、DPO)、推理层(AST 代码切片、RAG 检索、LLM 推理)、评估层(严格/宽松召回率、Wilson 置信区间、消融对照、错题闭环)。

**第 4 章 零样本推理基线实验。** 报告 exp_01~exp_05 五个实验:能力摸底、传统工具对比、RAG 知识库增强、难样本压力测试与消融、Prompt 工程消融。本章回答问题 1 与问题 2。

**第 5 章 网络安全专用模型训练。** 报告 exp_06 的 Phase 1~5 训练实验:Phase 1 lr sweep、Phase 2 r=32 增容失败、Phase 3 KnItLM CPT 突破、Phase 4 Prompt Distillation、Phase 5 DPO 边界校准。本章回答问题 3 与问题 4。
