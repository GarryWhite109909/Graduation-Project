## 五、必须手工完成的任务（不能依赖自动化）

1. **样本标注（Ground Truth）**：8 列表格（样本 ID、路径、CWE、风险等级、是否漏洞、触发条件、Source→Sink 污染路径、修复思路）
2. **RAG 漏洞知识库内容编写**：30-50 条，每条含 漏洞定义/代码特征/危险函数/安全示例/常见坑/修复方案
3. **Prompt 工程**：零样本 / Few-shot / 思维链 三套对比
4. **难样本设计**：假过滤、跨文件污点、真实 CVE 片段，占总样本 20-30%
5. **修复建议质量评分**：5 维度（语法/是否修复/是否引入新问题/可维护性/解释匹配），满分 10
6. **实验结论分析**：用"现象→原因→启示"三段论写为什么
