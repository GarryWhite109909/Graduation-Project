### 已完成（2026-06-28）：第一阶段摸底

1. 创建 `experiments/exp_01_basic_scan/` 目录。

3. 编写 `run_experiment.py`：读取样本 → 调用 Ollama API → 统一 Prompt → 记录耗时与判定 → 输出 `results.json`。

1. 安装 Bandit（Python 静态分析）与 Semgrep（多语言）。
2. 在 `experiments/exp_02_baseline_tools/` 下编写统一对比脚本，复用第一阶段 14 段样本。
3. 用同一批样本分别跑 Bandit / Semgrep，记录检出、漏报、误报、耗时。

5. 同步补充 3-5 段"难样本"（绕过式过滤、跨文件污点、真实 CVE 片段），观察 LLM 与传统工具在非典型漏洞上的差异。
6. 整理第二份实验报告 `exp_02_report.md`。

## 六、备注

- 先不要搭前后端框架，需求会在实验后明确。
- 所有实验过程和 Prompt 迭代都要保留，后续写论文直接可用。
- 模型名称注意核实：Ollama 中实际可用的模型名要与代码里配置的一致。
- **显存管理约定**：每次实验脚本跑完必须主动从显存卸载模型（Ollama `keep_alive=0`），
  只有在用的时候才加载。以后可能需要拉取多个模型对比，不卸载直接跑另一个会爆显存。
  `run_experiment.py` 已默认在末尾卸载，如需保留加 `--keep-loaded`。
