## 五、下一步具体行动（按优先级）

### 1. 高优先级：扩充难样本集（本周内）

这是当前最大短板。exp_01 报告自己承认"14 个样本统计意义有限，100% 仅说明下限"。
没有难样本，exp_03 的 RAG 价值无法证明（典型样本上 RAG 不提升准确率）。

具体做法：
1. 在 `experiments/exp_04_hard_samples/samples/` 新建目录
2. 按类型准备 20-30 段难样本：
   - SQL 注入：假过滤（`replace("'","")`）、二次注入、盲注
   - XSS：过滤后绕过（大小写、编码、事件处理器）
   - 命令注入：黑名单绕过、`$()` 替代反引号
   - 路径穿越：双重编码、`..%2f` 绕过
   - 真实 CVE：从 CVE PoC 仓库摘 5-10 段
   - OWASP WebGoat：摘 5 段典型题
3. 为每个样本写 ground truth（8 列标注表，见 docs/必须手动学习的地方.md）
4. 重跑 exp_01/exp_02/exp_03，对比典型样本与难样本的差距

### 2. 中优先级：Prompt 工程对比

设计 3 套 Prompt 在同一批难样本上对比：
- 零样本（当前版本）
- Few-shot（带 2-3 个典型漏洞示例）
- 思维链（要求模型先逐步分析再下结论）

### 3. 中优先级：AST 切片

用 tree-sitter 把长文件按函数切分，解决上下文窗口与注意力衰减。

### 4. 低优先级：系统 MVP 开发

待实验阶段结论稳定后再开始，避免需求反复推翻。
