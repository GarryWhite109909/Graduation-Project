### 专项：DeepSeek 安全样本优化（2026-06-30 失败）

> **目标**：将 deepseek-coder-v2:16b 安全样本误报率从 100% 降到 ≤10%，保持召回率 100%、准确率 ≥92.9%。
> **实际结果**：**失败**。各种工程手段均无法从根本上解决 deepseek 的知识盲区问题，最终的 100% 准确率全靠后处理 `safe_override` 硬覆盖，并非模型自身能力提升。
>
> **失败原因**：
> - deepseek-coder-v2:16b 体量大（16B MoE），存在根本性知识盲区（误判 `?` 占位符为拼接、误信 `subprocess.run` shell 默认 True 等）
> - 纯 Prompt 工程（5184 字符）无法纠正其判定逻辑，反而使耗时翻倍（9.6s → 26.8s）
> - RAG 知识增强受限于 embedding 语义相似度偏向漏洞描述，检索到的仍是漏洞知识而非安全模式知识，反而强化了误判
> - 后处理白名单兜底虽然能将 FPR 降至 0%，但这是外部规则覆盖而非模型能力改进，且维护成本高、泛化性差
> - **结论**：16B MoE 模型不适合作为安全专用模型优化基座。体积大、知识盲区深、工程手段效果有限。后续安全专用模型将以 qwen2.5-coder:7b 为基座。

  - **默认主模型已切换为 qwen2.5-coder:7b**，并作为后续安全专用模型优化基座
- [✓] **Prompt 精简实验发现**：
  - 完整版 SYSTEM_PROMPT（5184 字符）使 deepseek 耗时从 9.6s 涨至 26.8s（+180%），但纯 prompt 下 deepseek 安全样本误报率仍为 100%，准确率未因 prompt 增加而提升
  - 对 qwen7b 而言，完整版 prompt 是其达到 100% 准确率的关键：
    - 完整版：recall=100%、FPR=0%、accuracy=100%
    - 超精简版（移除安全模式白名单）：recall=100%、FPR=50%、accuracy=92.9%（safe_02 误判）
    - 中等精简版（保留白名单、移除 Few-shot 与硬编码规则）：recall=83.3%、FPR=0%、accuracy=85.7%（command_injection_01、path_traversal_01 漏报）
  - **结论**：Few-shot 示例与硬编码凭证规则对 qwen7b 的召回率有重要作用，不能简单删除；当前保留完整版 SYSTEM_PROMPT 作为默认
- [ ] ~~LoRA/QLoRA 微调 deepseek-coder-v2:16b~~ **已取消**（因模型体量大、知识盲区深，放弃以 deepseek 为安全专用模型基座，后续改为对 qwen2.5-coder:7b 进行微调）
